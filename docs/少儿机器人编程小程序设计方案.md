## 产品概述

### 定位
一款面向 6-12 岁儿童的 **AI 对话式机器人设计小程序**，小朋友通过和 AI 聊天，就能设计出自己的机器人，并预览动画效果。

### 核心价值
| 项目 | 说明 |
|------|------|
| **目标用户** | 6-12 岁儿童及家长 |
| **使用场景** | 家庭娱乐、编程启蒙、创意激发 |
| **核心亮点** | 自然语言交互 + 3D 实时预览 |
| **硬件要求** | 无需实体机器人，纯虚拟模拟 |

### 用户体验流程
```
小朋友：我想做一个会跳舞的机器人
   ↓
🤖 AI：好的！它有轮子还是腿？什么颜色？
   ↓
小朋友：有轮子，红色的，手臂会摆动
   ↓
🤖 AI：生成中... ✨
   ↓
📱 屏幕显示 3D 机器人 + 跳舞动画预览
```

## AI 设计

### AI Prompt 模板

```
你是一个少儿机器人设计助手，和 6-12 岁小朋友对话。

## 角色设定
- 名字：机器人设计师小蝌
- 语气：亲切、活泼、鼓励式
- 表情：多用 emoji 🤖✨🚀

## 对话任务
1. 欢迎小朋友，引导描述想要的机器人
2. 追问关键特征：
   - 外观：颜色、形状、大小
   - 结构：轮子/腿、手臂、头部
   - 功能：会做什么动作
3. 确认设计，输出结构化数据

## 输出格式
设计完成后，输出 JSON：
{
  "name": "机器人名字",
  "description": "机器人描述",
  "parts": {
    "base": "轮子/腿",
    "arms": "有/无",
    "head": "圆形/方形"
  },
  "colors": ["主色", "辅色"],
  "actions": ["走路", "挥手"],
  "model_config": {
    "scale": 1.0,
    "materials": {...}
  }
}

## 注意事项
- 用简单易懂的语言
- 多鼓励，多肯定
- 每次只问 1-2 个问题
- 不要一次问太多
```

## 数据库设计

### 数据表结构

```sql
-- 用户表
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) UNIQUE NOT NULL,
  nickname VARCHAR(50),
  avatar VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_openid (openid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 作品表
CREATE TABLE robots (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  parts JSON NOT NULL, -- {base, arms, head, ...}
  colors JSON NOT NULL, -- ["红色", "蓝色"]
  actions JSON NOT NULL, -- ["走路", "跳舞"]
  model_config JSON, -- 3D 模型配置
  thumbnail VARCHAR(255), -- 缩略图 URL
  is_public TINYINT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  INDEX idx_user (user_id),
  INDEX idx_public (is_public)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 对话记录表
CREATE TABLE conversations (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  robot_id BIGINT,
  role ENUM('user', 'assistant') NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (robot_id) REFERENCES robots(id),
  INDEX idx_user (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 动作库表（预设动画）
CREATE TABLE actions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL,
  display_name VARCHAR(50) NOT NULL,
  emoji VARCHAR(10),
  animation_url VARCHAR(255), -- 动画文件 URL
  description VARCHAR(255),
  sort_order INT DEFAULT 0
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## API 接口设计

### 核心接口

```
# AI 对话
POST /api/ai/chat
Body: { user_id, message, conversation_id }
Response: { reply, robot_design_json }

# 保存作品
POST /api/robots
Body: { user_id, name, parts, colors, actions, model_config }
Response: { robot_id }

# 获取作品列表
GET /api/robots?user_id=xxx&page=1
Response: { list, total }

# 获取作品详情
GET /api/robots/:id
Response: { robot_detail }

# 删除作品
DELETE /api/robots/:id
```
