## 1. 项目概览

**Web System** 是一个全栈 Web 应用平台，包含管理后台、少儿教育门户、微信小程序和配套后端微服务。项目采用 Rush monorepo 架构管理多应用、多服务的代码和依赖。

```
web_system/
├── apps/                      # 前端应用
│   ├── admin-web/            # 管理后台 (Vue3)
│   ├── portal/               # 少儿教育门户 (Vue3)
│   └── mini-app/             # 微信小程序
├── servers/                   # 后端服务
│   ├── gateway/              # API 网关 (NestJS，端口 3000)
│   ├── auth-service/         # 认证服务 (NestJS，端口 3001)
│   ├── user-service/         # 用户服务 (NestJS，端口 3002)
│   └── ai-service/           # AI 服务 (NestJS，端口 3003)
├── packages/                  # 共享包
│   ├── types/                # TypeScript 类型定义
│   └── shared/               # 共享工具函数
├── common/                    # 公共配置文件
└── docs/                      # 技术文档
```

## 3. 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端框架** | Vue 3 + TypeScript | SPA 应用 |
| **UI 库** | Ant Design Vue | 企业级组件库 |
| **构建工具** | Vite | 快速开发与构建 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **路由** | Vue Router | SPA 路由 |
| **后端框架** | NestJS | 企业级 Node.js 框架 |

| **API 代理** | http-proxy-middleware | 请求转发中间件 |
| **数据库** | MySQL + TypeORM | 生产环境持久化 |
| **缓存** | Redis | 会话/令牌缓存 |
| **认证** | JWT (Passport) | 无状态认证 |
| **包管理** | pnpm + Rush | Monorepo 依赖与构建管理 |
| **进程管理** | PM2 | 生产环境进程守护 |
| **反向代理** | Nginx | SSL 终止、域名路由、静态资源 |
| **微信小程序** | UniApp | 跨平台小程序框架 |

## 5. 网关设计

Gateway 是整个系统的统一入口，负责三类请求的分发：

| 分类 | 前缀 | 职责 |
|------|------|------|
| 🔍 健康检查 | `/health` | 服务存活性检测 |
| 🔄 API 代理 | `/api/*` | 转发到后端微服务 |
| 📦 静态资源 | `/static/*` | SPA 前端文件服务 |

三类 URL 前缀互不冲突，未匹配的 `/api/*` 返回 404 而非静默转发。完整的路由映射、代理规则、路径重写等细节见独立文档：

> 📄 [网关 URL 规划](./网关URL规划.md)

## 6. 后端微服务

### 6.1 Auth Service（认证服务）— 端口 3001

- 用户注册/登录
- JWT Token 签发与验证
- 权限校验
- 依赖：MySQL、Redis、JWT Secret

### 6.2 User Service（用户服务）— 端口 3002

- 用户 CRUD（列表、详情、创建、更新、删除）
- 用户状态管理
- 依赖：MySQL

### 6.3 AI Service（AI 服务）— 端口 3003

- AI 对话与生成能力
- 依赖：AI API（外部服务）

## 9. 部署架构

### 9.1 生产部署

**双服务器架构**：

| 服务器 | IP | 角色 |
|--------|-----|------|
| 69 服务器 | 42.194.200.69 | SSL 终止、反向代理（Nginx） |
| 246 服务器 | 106.52.176.246 | 应用服务器（Nginx + Gateway + 微服务 + MySQL + Redis） |

**请求链路：**

```
用户 → HTTPS → 69:Nginx(SSL) → HTTP → 246:Nginx → Gateway/静态资源 → 微服务
```

**进程管理**：PM2 管理 Gateway、Auth Service、User Service 三个 Node.js 进程。

**部署流程：**

1. 本地构建：`pnpm build`（构建前端 + 后端）
2. 上传静态资源：`rsync` 到 246 服务器的 Nginx 静态目录
3. 上传服务器代码：`scp` 后端 dist 到 246
4. 重启服务：`pm2 restart all`

### 9.2 本地开发

一键启动脚本 `start-dev.sh`：

```bash
./start-dev.sh
```

按顺序启动：
1. Auth Service (:3001)
2. User Service (:3002)
3. Gateway (:3000)
4. Admin Web (:5173)
5. Portal (:3004)

## 10. 配置管理

### 10.1 Gateway 环境变量

文件：`servers/gateway/.env`（开发）/ `.env.local`（生产）

```bash
PORT=3000                           # 服务端口
HOST=0.0.0.0                        # 监听地址
JWT_SECRET=dev-secret-key-12345     # JWT 密钥
JWT_EXPIRES_IN=7d                   # Token 有效期
CORS_ORIGINS=*                      # 跨域白名单
AUTH_SERVICE_URL=http://localhost:3001   # 认证服务地址
USER_SERVICE_URL=http://localhost:3002   # 用户服务地址
AI_SERVICE_URL=http://localhost:3003     # AI 服务地址
STATIC_ROOT=./public                # 静态资源目录
LOG_LEVEL=info                      # 日志级别
```

### 10.2 PM2 配置

文件：`ecosystem.config.js`

管理 Gateway、Auth Service、User Service 三个进程，配置日志轮转、内存限制、自动重启。
