# 管理后台升级 · 需求文档

## 背景

当前 admin-web 功能简陋：顶部横排导航、Dashboard 只有占位文字、菜单"系统设置"之前无路由、无权限控制。需要升级为标准的后台管理系统。

## 用户故事

- 作为**管理员**，我希望能看到**数据看板**，以便了解平台运营概况
- 作为**管理员**，我希望能使用**左侧菜单**导航，以便快速切换功能
- 作为**管理员**，我希望能**控制角色权限**，以便不同管理员看到不同菜单
- 作为**管理员**，我希望能通过**系统设置**控制功能开关，以便灵活运营

## 验收标准（EARS）

### 左侧菜单
- When 登录成功后，左侧栏应显示可折叠的菜单树
- When 点击菜单项，右侧内容区应显示对应页面
- While 某个菜单项无权限时，应不显示该项

### 看板仪表盘
- When 进入首页，应显示 4 个统计卡片（用户总数、今日活跃、课程数、系统日志数）
- When 加载数据时，卡片应显示 loading 骨架，加载完成后展示数字

### 权限系统
- When 角色为 viewer，应看不到"系统设置"和"用户管理"菜单
- When 角色为 editor，应能看到但不可删除用户
- When 角色为 admin，应拥有全部权限
- When 未来需要按钮级权限时，权限模型应支持扩展

### 系统设置
- When 修改基本设置并保存，应在 1 秒内返回成功
- When 关闭"微信扫码登录"开关，Portal 登录页应隐藏该 Tab
