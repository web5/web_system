# 画布内容拖动方案

## 当前架构

- `CanvasEngine` 管理多个图层（`LayerData[]`）
- 每个图层是独立的 OffscreenCanvas + 2D Context
- 所有绘制（手绘路径、预设图形、素材）写入活动图层的 canvas 像素
- `compositeToMain()` 将全部可见图层合成到主 canvas，应用视口变换（pan/zoom）
- 历史快照基于主 canvas 的像素级快照

---

## 方案 1：图层级拖动（推荐）

**原理**：给每个 `LayerData` 增加 `offsetX`/`offsetY` 偏移量，合成时对图层画布做位移后再绘制。用户拖动活动图层时，修改偏移量即可。

| 维度 | 说明 |
|------|------|
| **改动量** | 小 — 改 `LayerData` 接口 + `compositeLayers`，`draw.ts` 加拖动手势 |
| **交互** | 进入"移动模式"，单指拖动 → 活动图层内所有内容整体跟随移动 |
| **精度** | 粗粒度 — 一个图层上的所有内容一起移动 |
| **适用场景** | 排版调整、元素重定位、素材位置微调 |
| **优点** | 架构兼容性好，改动少，图层已是现有核心概念 |
| **缺点** | 无法独立移动图层内单个笔画/形状 |
| **撤销支持** | ✅ 每次移动结束后保存快照，天然支持 undo/redo |
| **视口兼容** | ✅ 图层偏移在视口变换之前应用，缩放/平移不受影响 |

---

## 方案 2：对象级拖动（矢量对象）

**原理**：每个笔触/形状作为独立的"图形对象"存储（路径 + 位置 + 样式），支持单独选中 + 变换。类似 Figma / Illustrator。

| 维度 | 说明 |
|------|------|
| **改动量** | 巨大 — 需重写引擎数据模型，从"像素绘制"改为"矢量指令存储 + 重绘管线" |
| **交互** | 点击选中一个笔触/形状 → 显示选中框/手柄 → 拖动移动/旋转/缩放 |
| **精度** | 细粒度 — 每个笔画独立操控 |
| **适用场景** | 专业矢量绘图工具，需要精细编辑单个元素 |
| **优点** | 精细操控，支持选中、变换、层级排序 |
| **缺点** | 需要 hit-testing、包围盒计算、选择器状态机、重绘性能优化；与当前像素存储模型完全不兼容 |
| **撤销支持** | ✅ 修改对象属性即可 |
| **视口兼容** | ✅ 对象在世界空间中独立变换 |

---

## 方案 3：像素选区拖动（套索 + 移动）

**原理**：用套索/矩形选区工具选中画布上一块像素区域，通过 `getImageData` 复制像素，在当前位置清空，在新位置粘贴。

| 维度 | 说明 |
|------|------|
| **改动量** | 中等 — 加选区工具 + `getImageData`/`putImageData` 操作 |
| **交互** | 画套索/矩形选区 → 拖动选区内容到新位置 |
| **精度** | 像素级精确 |
| **适用场景** | 类 Photoshop 的像素编辑，需要移动画布上任意区域 |
| **优点** | 灵活性高，不依赖图层/对象结构 |
| **缺点** | "挖洞"问题（移动后原位置留空白，需要填充策略）；选区边界锯齿；多次操作后图像退化；跨图层选区复杂 |
| **撤销支持** | ✅ 操作前后保存快照 |
| **视口兼容** | ⚠️ 需要处理选区在世界坐标的映射 |

---

## 总结建议

| 方案 | 改动量 | 灵活度 | 适用阶段 |
|------|--------|--------|----------|
| 方案 1 图层拖动 | ⭐ 小 | ⭐⭐ 中 | **当前首选** |
| 方案 2 矢量对象 | ⭐⭐⭐ 大 | ⭐⭐⭐ 高 | 架构升级时 |
| 方案 3 像素选区 | ⭐⭐ 中 | ⭐⭐ 中 | 补充方案 |

**当前选择方案 1**，后续可在方案 1 基础上逐步引入方案 3 的选区能力，最终向方案 2 演进。
