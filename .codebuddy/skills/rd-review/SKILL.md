# 🔍 Review Agent（代码审查）

## 职责

在 execute 完成后自动自检代码质量，确保无遗漏。

## 触发条件

- execute Agent 完成所有 TODO
- 用户说"审查"、"检查"、"review"
- 较大改动完成后

## 工作流

```
代码变更完成
  ↓
1. read_lints → 检查所有修改文件
  ↓
2. 运行受影响的测试 → 确认全绿
  ↓
3. 人工检查项：
   - [ ] console.log 已删除
   - [ ] 导入无遗漏
   - [ ] 错误处理完整
   - [ ] 类型定义正确
  ↓
4. 边界条件检查：
   - [ ] 空值/null 处理
   - [ ] 权限校验
   - [ ] 并发/竞态
  ↓
输出报告
```

## 输出格式

```markdown
## 代码审查报告

### Lint
- ✅ 通过 / ⚠️ N 个警告

### 测试
- ✅ N/N 通过

### 人工检查
- [x] console.log 清理 ✅
- [x] 导入完整 ✅
- [ ] 错误处理：缺少对 xxx 的处理 ⚠️

### 边界条件
- [x] 空值处理 ✅
- [x] 权限校验 ✅

### 结论
✅ 通过 / ⚠️ 建议修复 2 项后通过
```

## 审查强度

| 改动规模 | 审查深度 |
|---------|---------|
| 单文件 bug fix | 快速自检（lint + 跑测试） |
| 新功能（3-10 文件） | 完整自检 + 边界条件 |
| 架构变更（10+ 文件） | 完整自检 + 逐步确认 |
