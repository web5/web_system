# 技术方案审查 (Tech Review)

## 何时触发

当用户给出或 AI 提出涉及以下任一方面的方案时，**自动启动审查**：

- 新建/拆分后端服务
- 数据库表设计或迁移方案
- API 接口设计（RESTful 路径、参数、响应格式）
- 前端架构变更（状态管理、路由设计、组件拆分）
- 部署方案（Docker、K8s、Nginx 配置）
- 安全相关（认证流程、权限模型、数据加密）
- 第三方服务选型（存储、消息、监控）
- 前端组件库/工具链选型

**不触发**：小 bug 修复、文案修改、单文件样式调整。

---

## 审查维度（腾讯 + 字节共同实践）

### 1. 架构合理性

| 检查项 | 腾讯实践 | 字节实践 |
|--------|----------|----------|
| 服务职责单一 | 微服务不超过 3 个核心职责 | 模块边界清晰，遵循单一职责 |
| 服务间通信 | 通过网关统一路由，不直连 | 使用标准化 API 契约（protobuf/thrift） |
| 数据库归属 | 每服务独享数据库，不跨服务 join | 数据库私有，通过 API 共享数据 |
| 公共逻辑 | 提取到 shared packages | 提取到 common 模块 |

**审查问题**：
- 这个服务是否职责单一？能否进一步拆分或合并？
- 是否需要新建表，还是可以复用现有表？

### 2. 安全设计

| 检查项 | 腾讯实践 | 字节实践 |
|--------|----------|----------|
| 认证 | JWT + 网关统一验证 | OAuth2.0 + 统一认证中心 |
| 授权 | RBAC 角色权限，最小权限原则 | 基于策略的访问控制 (ABAC) |
| 敏感数据 | 加密存储，脱敏展示 | 字段级加密 + 审计日志 |
| API 安全 | 限流、防重放、参数校验 | 网关层统一限流 + 参数白名单 |

**审查问题**：
- 是否需要登录？权限模型是什么？
- 敏感字段（密码、手机号）是否加密/脱敏？
- 是否有操作日志记录？

### 3. 数据设计

| 检查项 | 腾讯实践 | 字节实践 |
|--------|----------|----------|
| 表设计 | 遵循 3NF，适当冗余提升性能 | 按查询模式反范式设计 |
| 索引 | 关键查询字段必加索引 | 覆盖索引优先，避免回表 |
| 分页 | 游标分页（大表）或 offset（小表） | 统一游标分页 |
| 软删除 | 核心数据用 deleted_at 标记 | 用 status 字段标记 |

**审查问题**：
- 表结构是否满足 3NF？
- 高频查询字段是否有索引？
- 是否需要软删除？

### 4. 性能与扩展

| 检查项 | 腾讯实践 | 字节实践 |
|--------|----------|----------|
| 缓存 | Redis 热点数据缓存 | 多级缓存（本地 + 分布式） |
| 异步 | 消息队列处理耗时任务 | 事件驱动 + 异步任务 |
| 连接池 | 数据库连接池合理配置 | 连接池监控 + 自动扩缩 |
| CDN | 静态资源 CDN 加速 | 边缘计算 + CDN |

**审查问题**：
- 是否需要缓存？缓存策略是什么？
- 是否有慢查询风险？
- 是否支持水平扩展？

### 5. 可维护性

| 检查项 | 腾讯实践 | 字节实践 |
|--------|----------|----------|
| 监控 | 接入 TAM/蓝鲸监控 | 全链路追踪 (OpenTelemetry) |
| 日志 | 结构化日志 + CLS | 统一日志平台 |
| 配置 | 配置中心统一管理 | 环境变量 + 配置中心 |
| 灰度 | 按比例灰度发布 | 特性开关 (Feature Flag) |

**审查问题**：
- 是否有健康检查端点？
- 关键操作是否有日志？
- 配置是否可热更新？

## 输出格式

审查完成后，按以下格式输出：

```markdown
## 技术方案审查

### 方案概述
> 一句话描述

### ⚠️ 风险项（必须解决）
- [ ] 风险1：描述 + 建议

### 💡 优化建议（建议采纳）
- [ ] 建议1：描述

### ✅ 合理项（无需改动）
- 项1：理由

### 综合评分
架构: ⭐⭐⭐⭐ / 安全: ⭐⭐⭐ / 数据: ⭐⭐⭐⭐ / 性能: ⭐⭐⭐

### 结论
✅ 通过（建议优化后执行）/ ⚠️ 有条件通过 / ❌ 需重新设计
```
